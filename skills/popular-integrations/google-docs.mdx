## Quick Overview

Google Docs integration with Lindy enables your AI agents to become sophisticated document management assistants. The integration provides both Triggers (what can start a Lindy workflow) and Actions (what Lindy can do), making it perfect for automating document creation, editing, approval workflows, and content management.

This comprehensive integration transforms Lindy into a powerful document automation platform, capable of handling everything from simple document creation to complex template merging and dynamic content management.

Key benefits of Google Docs integration:
- **Automated document creation** – Generate docs from templates with dynamic data
- **Real-time content management** – Update and edit documents automatically
- **AI-driven editing** – Intelligent text processing and formatting
- **Approval workflows** – Streamlined document review and approval processes

## How to set up Google Docs

<Steps>
  <Step title="Add Google Docs to your workflow">
    Click on the plus sign at the bottom or right-click anywhere on the canvas and select "Add Trigger" or "Add Action" from the pop-up menu and choose "Google Docs"

  </Step>
</Steps>

## Google Docs Triggers

#### New Document Created 
<Frame>
  <img src="/lindy-brand-assets/google_docs_3.png" alt="Document Created trigger" />
</Frame>
Fires the moment a brand-new Google Doc appears in a chosen folder:
- **Folder filtering** - Optionally select specific folder, defaults to watching entire Drive
- **Dynamic payload** - Provides title, url, documentId, createdTime, creatorEmail, parentFolders
- Perfect for auto-formatting new docs, posting Slack notifications, kickoff approval workflows

#### New or Updated Document
<Frame>
  <img src="/lindy-brand-assets/google_docs_4.png" alt="Document Updated trigger" />
</Frame>
Fires on both creations and any subsequent modifications:
- **Real-time monitoring** - Detects text edits, images, style changes, etc.
- **Enhanced payload** - Includes lastModifiedTime, lastModifyingUser information
- Ideal for change-log summaries, content sync to CMS, maintaining documentation repos

<Tip>
  Google push notifications are near-real-time (5-15 seconds). For large organizations, segment by folder to avoid hitting daily push quotas.
</Tip>

## Configuration Best Practices

<AccordionGroup>
  <Accordion title="For Document Creation">
    Optimize your document generation workflows:
    - Begin complex flows with Get Document to confirm correct file
    - Use template-based creation for consistent formatting
    - Implement proper folder organization and naming conventions
    - Store document URLs (not IDs) for human accessibility
  </Accordion>
  
  <Accordion title="For Content Management">
    Set up robust document processing:
    - Combine Append, Replace, and Update actions thoughtfully
    - Use Conditions on triggers to route by document type
    - Insert delays for high-volume operations to respect quotas
    - Pre-cache images in Drive for faster replacement operations
  </Accordion>
  
  <Accordion title="For Integration Workflows">
    Create reliable document automation:
    - Plan for Google API rate limits (100 writes/min per user)
    - Ensure proper Drive permissions for Shared Drives
    - Implement error handling for document size limits (1.02M characters)
    - Use organization-managed Google Workspace for compliance
  </Accordion>
</AccordionGroup>

## Limits & Security Considerations

<AccordionGroup>
  <Accordion title="API Limitations">
    - **Rate Limits**: ~100 writes/minute per user before exponential back-off
    - **Document Size**: 1.02 million character maximum
    - **Image Requirements**: External URLs must be publicly accessible
    - **Trigger Quotas**: Google's daily push notification caps
  </Accordion>
  
  <Accordion title="Security & Compliance">
    - **Authentication**: OAuth2 with Drive + Docs scopes required
    - **Permissions**: Content manager access needed for Shared Drives
    - **Data Handling**: Lindy processes content transiently during execution
    - **Compliance**: HIPAA/SOC 2 compatible with managed Google Workspace
  </Accordion>
</AccordionGroup>
